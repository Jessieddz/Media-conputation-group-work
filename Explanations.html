Explanations

General Summary

  • All content blocks were grouped inside containers with visible colours and shadows, making it easy for readers to digest the information.
  • Each section was structured with semantic HTML elements like 'header', 'nav', 'main' and 'footer' to ensure clean document structure and readability.
  • A navigation bar was implemented to allow quick access to any section on the page, improving the user experience and usability of the webpages.
  • A carousel component was used to make the webpage more dynamic and space-efficient, helping to avoid the clutter of information.
  • Custom fonts were imported through google fonts to improve the aesthetics of the website allowing for a more readable page
  • A flexbox layout was applied to content areas an the navigation to allow viewing on different devices which may have differing sizes
  • Hover effects were implemented to create a more interactive and dynamic webpage which would in turn enhance user engagement through CSS animations.

HTML

  • Alt text: allows people unable to see the images on screen to still be able to digest and understand all the information being depicted.
  • Anchor : creates links to other web pages, to add fun and connectivity.
  • Div: Names and divides different parts of the content, giving them the same or different class names depending on needs, to make all of the information more organised.
  • Figure
    • Figure: Includes image and the caption of the image and lets it look like a whole section thus improving clarity within the webpage.
    • Image: adds an image to the web page, making it more fun and vivid.
    • Fig caption: show the sources, like where the picture is from.
  • List
    • Li: used to represent a listed item. Lists make the page clearer, so the audience can more easily understand and remember the key points.
    • ul: List different points in regular dot points (unordered, not numbered).
  • Line Break (br): adds a small line break between content. This can improve readability as it separates/spaces/divides bits of information.
  • Sections
    • Footer: References creators. It can also be a nice space to quote a slogan.
    • Header: includes a representative image that occupies most of the screen and shows the main idea of the webpage.
    • Main: The section in which we put the main body of information (that doesn’t belong in the header, footer, or nav bar) so the user can see all specific and important information within one big section
    • Navigation Bar: enables seamless movement between web pages. Each personal webpage features a fixed navigation bar, allowing viewers to easily enter and exit pages throughout the website. When selecting "game" or "sport," dropdown menus display representative icons and the individual page names, enhancing engagement and recognition. Additionally, the navigation bar includes a logo and hyperlinks, strengthening connections between various web pages.
    • Sections: clearly divides each category of information in a way that is understandable to the readers
  • Text
    • H1: used for the main sections of the web page, like the webpage’s name, and show the most important information for viewers
    • H2: used for sub-sections of the main heading, further organising the information
    • P: used for the main body text

CSS

  • Background
    • Background-color: Set the color of the background for a section, so they can easily be differentiated and the overall tone is more unified. For example, setting the same color scheme but having a different color background for each section can make viewers see different modules better, and setting background-color for the whole body to make the whole page more coordinated and integrated.
    • Background-image: Setting the background as a picture can increase interest and reflect the theme of the web page
    • Background-position: Determines the position of the image, which affects the distribution of the background image
    • Background-size: changes the background image’s size, such as the ones in the header.
  • Behaviours
    • Cursor: Sets the style of mouse cursor when it hovers over a certain element. This makes the webpage feel more interactive and engaging.
    • Scroll-behavior: Sets how the text is animated as it moves within a scrollable file, whether it is a smooth transition or a direct jump.
    • Transition: Allows an element to switch between two states when it is interacted with. This builds a more enjoyable and engaging experience for the user as the website will react to their movements.
  • Positioning
    • Align-items: Sets the vertical alignment or the position of the element
    • Gap: Sets size of gutters between rows and columns.
    • Justify-content: Sets the horizontal distribution of elements, such as flex-end, can be set according to the needs of each section
    • Margin: sets the spacing outside of the defined borders or parameters; spreads the text in a way that is readable and pleasant to look at.
    • Padding: Adds space between the borders of a parent element and its child elements
    • Position: Sets the type of positioning method used for an element; whether it is static, relative, fixed, absolute or sticky.
    • Text-align: decides the text’s position – such as left, right, or center – in order to match other contents well
  • Section Formatting
    • Border-radius: Makes the border's corners round to make the page look more fun and inviting.
    • Box-shadow: Sets a shadow effect for an element, such as a section or a button, in order to highlight it
  • Flex-box
    • Display: Adjusts a flex-box to create better and more flexible layout adjustments.
    • Flex-direction: The setting is vertical layout or horizontal layout, which can be set according to different needs
    • Flex-wrap: Sets whether flex-items must stay within their lines or are able to move beyond. Helps the adaptability of the webpage as it can be coherent on any size screen size.
    • “overflow: auto”: Adds scroll bars to the text when necessary (e.g. when the text has overflowed out of the borders of its sections.
    • Z-index: Sets the positioning of an element that is overlapping or being overlapped by other elements. Overlapping elements like this can make the website feel more dynamic and less rigid.
  • Sizing
    • Height: sets the height for elements, To better match the height of the other content
    • Width: sets the width for elements, To better match the width of the other content
    • Min-width: Sets the minimum width to prevent elements from becoming too small
    • Max-width: Sets the maximum width to prevent child elements from going beyond the borders of their intended margins.
  • Typography
    • Color: Sets the font color. This can be used to make the text better match the overall theme or to emphasise certain words.
    • Font-family: Different fonts can better match the theme.
    • Font-size: Sets the size of the text to make it more readable or to emphasise certain parts.
    • Font-weight: used to emphasise certain words
    • Line-height: Controls the line spacing of the text, so that the audience can read the text easily, and make the overall web page look more balanced and clean
    • Text-decoration: Sets what style certain lines of text should have.
    • Text-shadow: Sets a shadow effect onto text, which can be used to highlight its importance, such as for a web heading.
    • List-style: formats unique dot points at the start of the list to create a more engaging user experience.
